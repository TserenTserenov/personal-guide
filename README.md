# Персональное руководство

Это твоё персональное руководство по развитию в IWE. Репозиторий - контейнер, куда система (Портной) складывает занятия, а ты их проходишь и закрываешь. На твоих результатах система учится и делает следующие занятия точнее.

Этот файл (README) описывает сам репозиторий и не трогается автоматикой - он для тебя и для того, кто откроет репозиторий впервые.

## Как это работает за один проход

1. Утром система собирает занятие дня и кладёт его в `lesson/`.
2. В начале недели собирает недельное руководство (где ты, тема недели, задание) - это файл недели в `lesson/`.
3. Ты открываешь занятие (через IWE командой `/lesson` или просто читаешь файл) и выполняешь.
4. Закрываешь занятие (`/lesson-close`) - отметка о выполнении уходит в систему, обновляет твой цифровой двойник, и завтрашнее занятие это учитывает.
5. В конце недели вся папка `lesson/` целиком уезжает в архив `history/`, а в `lesson/` появляется новая неделя: файл недели и первый день.

## Структура репозитория

| Путь | Что это | Кто пишет |
|------|---------|-----------|
| `README.md` | этот файл, про репозиторий | вручную (не автоматика) |
| `profile.md` | твой профиль развития (ступень, узкое место) | система |
| `worldview.md` | мировоззренческая линза (фаза, мем, роли) | система |
| `methods.md` | методы под твоё узкое место | система |
| `guide.json` | служебный индекс для веб-страницы | система |
| `lesson/` | ТЕКУЩАЯ неделя: файл недели + до 7 дневных файлов | система |
| `workbook/` | твои ВЫПОЛНЕННЫЕ занятия (отметка + время + заметки) | ты, через `/lesson-close` |
| `history/` | архив по годам и месяцам: прошлые недели, дни, рефлексии | система + ты (рефлексии) |

### Папка `lesson/` - только текущая неделя

```
lesson/
  2026-W25.md      ← файл недели (где ты, тема и задание недели)
  2026-06-16.md    ← занятие дня (до 7 файлов за неделю)
  ...
  2026-06-22.md
```

В конце недели вся `lesson/` целиком переезжает в `history/<год>/<месяц>/`, и в `lesson/` остаётся новая неделя: файл недели плюс первый день.

### Папка `history/` - архив по годам

```
history/
  2026/
    06/
      2026-W25.md               ← заархивированная неделя
      2026-06-16.md ...          ← её дни
      reflection-2026-06-18.md   ← твои рефлексии
```

## Что трогать руками, а что нет

- **Не трогай руками:** `profile.md`, `worldview.md`, `methods.md`, `guide.json`, файлы в `lesson/`. Они пересобираются автоматически, ручные правки затрутся.
- **Твоё (через IWE):** закрытие занятия (`/lesson-close` пишет в `workbook/`), рефлексии в `history/`.
- Хочешь поправить содержание занятия - не правь файл, а дай обратную связь: она влияет на следующую сборку. Или попроси пересобрать.

## Где смотреть руководство - три способа

1. **Прямо в гите / VS Code** - открываешь файлы `lesson/` как есть.
2. **Через IWE** - команда `/lesson` открывает занятие дня и ведёт по нему.
3. **На сайте** - если подключить репозиторий к guide.system-school.ru (приложение «Aisystant Personal Guide»), руководство видно как веб-страница.

## Тайный или явный репозиторий

- **Тайный (приватный):** руководство видишь только ты. Подходит, если в нём личные данные. Веб-страница работает через приложение с доступом.
- **Явный (публичный):** руководство можно показать другим по ссылке на гит.

Выбор твой, система работает в обоих режимах. Личные данные всегда остаются у тебя; на сайт уходит только то, что ты подключил.

## Табло (панель показателей)

Табло **не собирает данные в этом репозитории**. Данные считаются и хранятся в системе (Neon): множитель и поток - ночным сбором, баллы и ступень - читаются вживую при сборке. В репозиторий приходит только готовый блок-снимок, который вставляется в занятие дня. То есть репозиторий - место показа, а не место сбора.

## Как лучше работать - коротко

- Утром: `/lesson` - открыть занятие.
- Сделал задание (45-90 мин) - `/lesson-close`.
- Раз в неделю смотри файл недели и сверяй тему и задание.
- Не редактируй сгенерированное; влияй на него обратной связью.
