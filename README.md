# Персональное руководство

Это твоё персональное руководство по развитию в IWE. Репозиторий - контейнер, куда система (Портной) складывает тексты для изучения и дневные табло, а ты их проходишь и закрываешь. На твоих результатах система учится и делает следующие тексты точнее.

Этот файл (README) описывает сам репозиторий и не трогается автоматикой - он для тебя и для того, кто откроет репозиторий впервые.

## Два потока: содержание и форма

- **Содержание (`guide/`):** тексты для изучения. Дневной и недельный файлы с фрагментами универсальных руководств, адаптированными под твою квалификацию, ступень и активные проекты. Это то, что ты читаешь и разбираешь.
- **Форма (`panel/`):** твоё состояние. Ступень, узкое место, ритм, метрики, рабочая повестка, слоты дня. Это данные, не текст для чтения.

Разделение намеренное: когда открываешь руководство - читаешь содержание. Когда смотришь на цифры - открываешь панель.

## Три уровня времени

- **Месяц (корень):** медленный тренд - когда что менялось (квалификация, ступень). Дописывается сверху, чтобы пролистать всю историю.
- **Неделя (`guide/<год>-W<номер>.md`):** тема недели, ключевое различение, задание на неделю. Только содержательный текст.
- **День (`guide/<дата>.md` + `panel/<дата>.md`):** текст для изучения сегодня (содержание) и снимок состояния (форма).

## Как это работает за один проход

1. Утром система собирает текст дня и снимок табло, кладёт в `guide/` и `panel/`.
2. В начале недели собирает недельный файл с темой и заданием недели.
3. Ты открываешь руководство (через IWE командой `/guide` или просто читаешь файл), изучаешь текст и дописываешь результат в тот же файл дня.
4. Закрываешь занятие (`/guide-close`) - проставляется отметка и время, результат уходит в систему и обновляет твой цифровой двойник.
5. Когда меняется крупное (получил квалификацию, перешёл на ступень) - система дописывает строку сверху в тренд `milestones.md`.
6. В конце недели весь недельный пакет (`guide/` и `panel/`) уезжает в архив `history/`, а в `guide/` появляется новая неделя.

## Структура репозитория

| Путь | Что это | Кто пишет |
|------|---------|-----------|
| `README.md` | этот файл, про репозиторий | вручную |
| `guide.json` | служебный индекс для веб-страницы (ссылки на guide/) | система |
| `milestones.md` | тренд развития: когда что менялось (квалификация, ступень), новое сверху | система дописывает сверху; ты тоже можешь |
| `guide/` | содержание: тексты для изучения (недельный файл + дни) | система + ты (низ файла дня) |
| `panel/` | форма: дневные снимки состояния (метрики, повестка, слоты) | система |
| `history/` | архив по годам и месяцам | система + ты (рефлексии) |

### Содержание - `guide/`

```
guide/
  2026-W26.md      ← недельный текст: тема, различение, задание
  2026-06-27.md    ← дневной текст: фрагмент руководства под твой контекст
  ...
```

Недельный файл задаёт контекст: что изучаем, почему именно сейчас, какой вопрос проверяет понимание. Дневной файл - конкретный фрагмент для работы в этот день.

### Форма - `panel/`

```
panel/
  2026-06-27.md    ← снимок состояния: ступень, метрики, узкое место, повестка, слоты
```

Данные состояния считаются и хранятся в системе (Neon); в репозиторий приходит только готовый дневной снимок. Репозиторий - место показа, не место сбора.

### Тренд развития - `milestones.md` (корень, месячный ритм)

Дописывается сверху, чтобы пролистать всю историю и увидеть, когда что менялось:

```
## 2026-06-17 - Ступень 5, квалификация Реформатор
## 2026-03-10 - Переход на ступень 4
## 2026-01-01 - Получил квалификацию X
```

### Архив - `history/`

```
history/
  2026/
    06/
      2026-W26.md            ← заархивированная неделя (содержание)
      2026-06-27.md          ← текст дня (содержание)
      2026-06-27-panel.md    ← табло того же дня (форма)
      reflection-2026-06-18.md
```

В конце недели весь пакет недели уезжает в `history/<год>/<месяц>/`, где файлы сортируются по дате и лежат рядом.

## Что трогать руками, а что нет

- **Не трогай:** `guide.json`, недельный файл, **верх файла дня (учебный текст)**. Пересобирается автоматически.
- **Твоё:** **низ файла дня** (что понял, заметки, рефлексия), закрытие занятия (`/guide-close`), свои вехи в `milestones.md`, рефлексии в `history/`.
- Хочешь поправить само задание - не правь файл, а дай обратную связь: она влияет на следующую сборку.

## Где смотреть руководство - три способа

1. **Прямо в гите / VS Code** - открываешь файлы `guide/` как есть.
2. **Через IWE** - команда `/guide` открывает руководство дня и ведёт по нему.
3. **На сайте** - если подключить репозиторий к guide.system-school.ru (приложение «Aisystant Personal Guide»), руководство видно как веб-страница.

## Тайный или явный репозиторий

- **Тайный (приватный):** руководство видишь только ты. Подходит, если в нём личные данные. Веб-страница работает через приложение с доступом.
- **Явный (публичный):** руководство можно показать другим по ссылке на гит.

Выбор твой, система работает в обоих режимах. Личные данные всегда остаются у тебя; на сайт уходит только то, что ты подключил.

## Как лучше работать - коротко

- Утром: `/guide` - открыть руководство дня.
- Изучил текст (45-90 мин), дописал заметки в тот же файл - `/guide-close`.
- Раз в неделю смотри недельный файл; раз в месяц пролистай `milestones.md`.
- Не редактируй сгенерированное; влияй на него обратной связью.
