# Дневной план на 2026-05-12 (понедельник, день 1)

## Контекст дня

Ты возвращаешься после паузы в 4 дня. Это нормально. Сегодня — облегчённый день, одна помидорка, простой элемент, поддерживающий нарратив. Цель не в достижении, а в том, чтобы вернуться в ритм.

Режим: Восстановление.

## Окно дня

**Время:** 10:00–10:30 (25 мин работа + 5 мин отдых)

**Место:** одно и то же, каждый день. Может быть:
- Любимое кафе, если работаешь оттуда.
- Свой стол дома, без уведомлений (телефон в другой комнате).
- Коворкинг, если там есть привычное место.

Место важно не как фетиш, а как якорь: мозг привыкает, что в этом месте и в этом время — 25 минут сосредоточения.

## Действие дня (выбери одно)

### Вариант A: Чтение (легче для восстановления)

1. Открой спецификацию, RFC или статью по твоему стеку. Примеры:
   - Go: https://golang.org/ref/spec (chapter on Interfaces)
   - Lua: https://www.lua.org/manual/5.4/ (Introduction)
   - Система: любая статья на habr.com или dev.to про твой основной язык.

2. Читай 10–15 минут (примерно две страницы или три раздела).

3. Напиши заметку (3–5 строк) в любом удобном месте (Notion, GitHub Gist, Obsidian, текстовый файл):
   - Что ты прочитал?
   - Один момент, который удивил или непонятен?
   - Примечание: где ты это мог бы применить в своей работе?

   Пример заметки:
   ```
   Тема: Go interfaces
   Прочитал про interface{} и type assertion.
   Непонятно: почему interface{} это не Object, как в Java?
   Применение: в моём REST API можно использовать interface{} для
   handle любых типов данных в функции unmarshalling.
   ```

4. Открой трекер (Toggl, Clockify или свой лог):
   - Название: "Reading: Go Spec Interfaces"
   - Время: 25 мин
   - Тег (если есть): #reading, #go
   - Статус: закрыто

5. После слота (5 мин отдыха): встань, выпей воды, не смотри в телефон.

### Вариант B: Кодирование (если хочется движения)

1. Открой свой pet-project (или возьми одну small task из работы, которую можно сделать за 25 мин).

2. Реализуй одну маленькую функцию или fix:
   - Добавь простую обработку ошибки.
   - Рефакторь один метод.
   - Напиши тест для одной функции.
   - Добавь логирование в одно место.

   Главное: это должно быть **одно конкретное действие**, не размытое "поработаю над проектом".

   Пример:
   ```go
   // Сегодня добавишь функцию для валидации email
   func validateEmail(email string) error {
     if len(email) < 5 {
       return fmt.Errorf("email too short")
     }
     // ...
     return nil
   }
   ```

3. Коммитнись:
   ```
   git add .
   git commit -m "add email validation"
   ```

4. Открой трекер:
   - Название: "Code: Email validation function"
   - Время: 25 мин
   - Тег: #code, #pet-project (или #work)
   - Статус: закрыто

5. После слота (5 мин отдыха): тот же ритуал.

## Критерий закрытия дня

**Минимум:** слот отмечен в трекере. Не важно, прочитал ли ты всю статью или закончил ли функцию. Важно: отметь, что ты прошёл 25 минут.

**Норма:** слот закрыт + заметка или коммит видны.

**Отлично:** всё выше + ты понимаешь, почему выбрал именно это действие (не потому, что я сказал, а потому, что это релевантно твоей работе).

## Вопрос для себя в конце дня

После того, как закроешь помидорку, ответь на один вопрос письменно (можешь в личный чат, можешь в заметку):

**«Слот закрыт / не закрыт — какое объяснение точнее описывает сегодня: ресурс, приоритет, или ритм?»**

Примеры ответов:
- «Ресурс: я был уставший после спринта, не смог сосредоточиться. Слот не закрыт».
- «Приоритет: срочная задача пришла в 10:15, пришлось перенести. Слот не закрыт, но понимаю почему».
- «Ритм: слот закрыт, условия позволили. Чувствую, что это вторая помидорка подряд, которая работает».

Этот вопрос не нужно записывать как вывод в философский трактат. Просто ответь себе в одном предложении. Ответ копится в памяти, и к концу недели в рефлексии ты увидишь паттерн.

## Если что-то помешает

**"У меня встреча в 10:00"** → перенеси слот на 14:00 или на завтра. Одна встреча не ломает ритм.

**"Я не знаю, что читать/писать"** → открой список:
- Читать: Go spec (Interfaces), Lua manual (Intro), Python docs (Data Model), любая статья про твой язык на habr/dev.to.
- Писать: возьми самую маленькую issue в своём pet-project. Если нет — напиши простую функцию для утилиты (генератор UUIDs, парсер CSV, что угодно).

**"Я пропустил день, теперь смысла нет"** → нет, смысл есть. Один день пропущен, это норма на этой ступени. Сегодня — новый день, новый слот. Ничего не ломается.
