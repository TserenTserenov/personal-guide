## Что изучить (10 мин)

### Фрагмент: Domain seed как мост между методом и работой

Ты пишешь руководства для других (WP-425, WP-426). Каждый ученик — Knowledge Worker из какого-то домена: разработчик, аналитик, архитектор. Методы одни, но примеры разные.

**Domain seed** — это заготовка, которая превращает общий метод в примеры конкретной работы:
- METHOD.003 (медленное чтение) для бэкенд-разработчика = чтение спец и RFC
- METHOD.004 (письмо) для аналитика = design docs и SQL-комментарии
- METHOD.005 (проговаривание) для архитектора = code review и архитектурные обсуждения

Без этого моста — методы повисают в воздухе, ученик не видит «это про мою работу».

**Таблица из seed выше:**
```
| Метод | Во что встраивается у Knowledge Worker |
| METHOD.003 | Встраивается прямо в работу: чтение RFC, спецификаций, чужого кода |
| METHOD.004 | Design docs, ADR, PR-описания, технические посты |
| METHOD.005 | Ревью кода и дизайнов, парное программирование, архитектурные обсуждения |
```

Вот этот маппинг — основа для WP-425 и WP-426.

Выбрано на основании: Active WP (WP-425, WP-426 в top активных), domain seed из поликорневого контекста, текущая область фокуса (WD).
